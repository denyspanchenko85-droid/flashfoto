​📸 FlashFoto — Розумний Фотобокс
​FlashFoto — це високопродуктивний Android-додаток для автоматизації процесу мобільної зйомки та миттєвого друку. Створений для роботи на івентах, де швидкість отримання готового результату (фото з брендованою рамкою) є критичною.
​Статус проекту: 🏗️ В розробці (Work In Progress). Активна фаза рефакторингу та впровадження прямого друку.
​⚡ Основний Сценарій (Workflow)
​Процес максимально автоматизований для мінімізації взаємодії з екраном:
​Запуск: Додаток автоматично відкриває камеру.
​Зйомка: Користувач робить фото standard-засобами камери.
​Обробка: Система автоматично визначає орієнтацію та накладає відповідний шаблон (Easter/Branding).
​Фіналізація: Результат зберігається в системну Галерею (Pictures/FlashFoto).
​Друк: Миттєва відправка на принтер (Silent Socket Printing або System Dialog).
​🛠️ Архітектурні Особливості (Atomic Design)
​Проект побудований на принципах Single Responsibility, що дозволяє легко замінювати окремі модулі (наприклад, перейти з Intent-камери на CameraX без переписування всього коду).
​WorkflowManager: Серце додатка. Координує весь ланцюжок від "сирого" фото до друку.
​PrintManager: Розумний диспетчер друку. Підтримує прямий друк через сокети (порт 9100) для обходу системних вікон.
​GalleryManager: Коректна робота з MediaStore для Android 10-13+.
​HistoryManager: Вбудована навігація по історії зроблених фото.
​CameraHandler: Ізольована логіка роботи з камерою та дозволами.
​SettingsManager: Гнучке керування конфігураціями (IP принтера, автодрук, збереження оригіналів).
​Logger: Автономна система логування помилок у файл із таймстампами.

​🛠️ Стек технологій
​Language: Kotlin
​Architecture: Modular / Atomic
​API Level: 24+ (Android 7.0+)
​Printing: Android Print Helper + Socket API (JetDirect/IPP)
​Storage: MediaStore API + FileProvider
​📝 Як користуватися (Для розробника)
​Проект збирається через Gradle.
​Додайте ваші шаблони в res/drawable під іменами easter_horiz_1 та easter_vert_1.
​Вкажіть IP-адресу вашого Epson-принтера в налаштуваннях (іконка шестерні).
​Переконайтеся, що принтер і смартфон знаходяться в одній мережі.
​Тінь, цей файл уже можна пушити в репозиторій. Він дає чітке розуміння, що проект серйозний.
